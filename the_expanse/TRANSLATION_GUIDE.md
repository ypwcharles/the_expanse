# 《无垠的太空/苍穹浩瀚》翻译工作指南

本指南旨在协助译者在现有的项目基础上，以最高效、最专业的方式贡献翻译，并确保最终生成的 EPUB 电子书具备一致的品质。

---

## 1. 项目资源概览

在开始翻译前，请熟悉以下目录结构：

*   **`original/`**: 存放所有英文原版 EPUB。这是翻译的唯一源头。
*   **`the.expanse.md`**: **核心译名表**。涵盖了所有出场人物、战舰、星系、官阶及科幻术语的统一译名。
*   **`Styles/stylesheet.css`**: 定义了最终电子书的字体、缩进、行间距等外观。
*   **`demo.xhtml`**: 每个章节在成书阶段需要嵌套的 HTML 模板。
*   **`中文电子书/`**: 查看当前已发布的译本参考样板。

---

## 2. 翻译工作流

推荐采用以下“三阶段”工作法：

### 第一阶段：准备工作
1.  **认领章节**：查看 `README.md` 中的进度表，确认未翻译的章节。
2.  **提取原文**：
    *   使用 Calibre 或 Sigil 打开原版 EPUB。
    *   找到对应章节的 XHTML 源码。
    *   将纯文本提取出来，或使用 OCR 辅助。
3.  **同步译名表**：在编辑器中分屏打开 `the.expanse.md`，随时准备检索词汇。

### 第二阶段：Markdown 编写 (草稿阶段)
在对应卷的目录下（如 `无垠的太空(7).../`）新建 `.md` 文件。
*   **标题格式**：`# 第X章 [视角人物]`（如 `# 第四章 霍顿`）。
*   **翻译要求**：
    *   **忠于原音**：角色对战、科学分析、内心独白需体现不同的语气。
    *   **术语严谨**：必须强制遵循译名表。如：PDC -> 定点防御炮；Crash Couch -> 抗压座椅。
    *   **译注使用**：遇到需要额外解释的背景，在文末添加 `### 译注` 段落。
*   **落款元数据**：
    ```markdown
    ---
    by [你的ID]
    译：YYYY.MM.DD
    校：YYYY.MM.DD
    ```

### 第三阶段：成品化 (集成阶段)
待章节经过初校后，需要将其整合进电子书结构：
1.  **套用模板**：复制一份 `demo.xhtml`，重命名为对应章节名（如 `th7_ch04.xhtml`）。
2.  **注入内容**：将 Markdown 的内容填入文件的 `div.section` 标签中。
3.  **维护 Styles**：如果某些排版 MD 无法完美表达（如特定的居中签名），可引用 `stylesheet.css` 中预设的类。

## 3. 技术规范建议

### 3.1 标点与排版
*   **全角标点**：正文统一使用中文全角标点。
*   **对话引号**：使用双引号 `“...”`，对话内的对话使用单引号 `‘...’`。
*   **数字/单位**：数字与中文之间建议空一格（如 `12 小时`），科学单位（如 `0.5 g`）保持英文。
*   **空行**：段落与段落之间请保留一个完整的空行。

### 3.2 译名维护规范
如果在翻译过程中遇到 `the.expanse.md` 中没有的新名词：
1.  **先行自译**：根据上下文和发音进行音译或意译。
2.  **记录待定**：在翻译稿中暂时加粗或标记，待全体校对时确认。
3.  **增补库**：确认无误后，将其增补至 `the.expanse.md` 的对应分类下，并注明首次出现的卷号及章节。

## 4. 推荐工具
*   **VS Code / Cursor**：强大的 Markdown 预览与全局搜索。
*   **Sigil**：最专业的 EPUB 源码编辑工具（处理 `demo.xhtml` 时必用）。
*   **The Expanse Wiki (Fandom)**：用于核实复杂的科技设定或地理关系。
*   **Pandoc**：用于将 Markdown 快速转换为 XHTML。
